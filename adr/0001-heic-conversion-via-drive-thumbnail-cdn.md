# ADR-0001: HEIC Image Conversion via Google Drive Thumbnail CDN

**Date:** 2026-04-11  
**Status:** Accepted  
**Deciders:** Chris Kirk, Seth McClintock

---

## Context

Chris shoots photos on iPhone 15 Pro Max, which saves images in HEIC/HEIF format. The Poshmark pipeline needs to:
1. Download photos from Google Drive
2. Analyze them via a vision AI (OpenAI-compatible API)
3. Post listings to Poshmark

The vision API only accepts JPEG, PNG, WebP, or GIF — HEIC is rejected with:
> "You uploaded an unsupported image. Please make sure your image has one of the following formats: ['gif', 'webp', 'jpeg', 'png']."

We evaluated several approaches to convert HEIC → JPEG:

## Options Considered

### 1. Sharp library (REJECTED)
Node.js image processing library. Fails because the server's pre-built Sharp binary lacks HEVC codec support:
> "Support for this compression format has not been built in (11.6003)"

### 2. FFmpeg with HEVC decode (REJECTED)
Server has ffmpeg 6.1 with `libheif-plugin-aomdec` installed. However, the HEIC files downloaded from Drive API are missing the `moov` atom required for proper HEIF decode — ffmpeg reports:
> "moov atom not found"

### 3. Google Drive Export API (REJECTED)
`files.export` endpoint with `mimeType=image/jpeg` would convert server-side, but Drive only supports export for Google Docs/Editors files — not user-uploaded images:
> "Export only supports Docs Editors files"

### 4. HEIF CLI tools (REJECTED)
Tools like `heif-convert` would work but add an extra system dependency with uncertain compatibility.

### 5. Drive Thumbnail CDN (ACCEPTED) ✅
Every file stored in Drive automatically gets a thumbnail generated by Google. The thumbnail URL is available in the file metadata and Drive's CDN serves it as a standard JPEG regardless of the original format — including HEIC. The thumbnail comes in multiple sizes (s220, s500, s1000, w1000, w2000-h2000) up to full resolution. We use the base URL (stripping the size suffix) to get the full-size JPEG.

**Key finding:** The base thumbnail URL (without `=s\d+` suffix) returns the full-resolution JPEG of the image, automatically converted by Google's infrastructure.

## Decision

Use the Drive thumbnail CDN (`thumbnailLink` from file metadata) to download photos as JPEG. The flow:
1. Call `files.get` with `fields=id,name,mimeType,thumbnailLink`
2. Strip `=s\d+` size suffix from `thumbnailLink` to get full-resolution URL
3. Download with service account OAuth token
4. Verify first bytes are JPEG magic number (`FF D8 FF`)
5. Save to temp file for vision analysis

## Consequences

**Positive:**
- No extra system dependencies (no ffmpeg HEIF plugins needed)
- Google's infrastructure handles all format conversions reliably
- Works for any image format Drive supports, not just HEIC
- Full-resolution output available

**Negative:**
- Extra API call per photo (one `files.get` to get metadata, one for download)
- Slight latency from CDN round-trip
- Depends on Drive service account having read access (already required)

## Implementation

```typescript
// In src/lib/drive.ts — downloadAndConvertPhoto()
const thumbnailUrl = photo.thumbnailLink.replace(/=s\d+$/, '');
const response = await axios.get(thumbnailUrl, {
  headers: { Authorization: `Bearer ${token}` }
});
const buffer = Buffer.from(response.data);
if (buffer[0] !== 0xff || buffer[1] !== 0xd8) {
  throw new Error(`Drive returned non-JPEG`);
}
```
